# 抽卡小遊戲需求書

## 專案概述

### 專案名稱
抽卡小遊戲

### 專案描述
這是一個為女朋友慶生而製作的抽卡小遊戲，透過抽卡的形式來揭示多樣生日禮物。遊戲僅參考 Pokemon Trading Card Game Pocket 的抽卡形式與動畫效果，風格、配色與主題完全獨立，待後續討論決定。

## 需求描述

### 參考遊戲分析：Pokemon Trading Card Game Pocket

#### 參考範圍說明
**僅參考抽卡形式與動畫效果，不參考風格、配色與主題**

#### 抽卡機制特點（參考部分）
- **開包形式**：每包包含一張卡牌
- **卡牌揭示**：卡牌有開包動畫效果與翻轉揭示動畫
- **視覺體驗**：注重開包過程的動畫與卡牌翻轉效果

#### 本專案參考重點
- 開卡包的視覺呈現方式（動畫流程）
- 卡牌揭示的動畫效果（翻轉、揭示時機）
- 卡牌排列與展示方式（多包時的排列）

#### 不參考的部分
- 寶可夢主題與風格
- 配色方案
- 卡牌視覺設計風格

### 功能需求

#### 頁面流程
1. **首頁/初始畫面**
   - 暫定：直接進入抽卡畫面
   - 後續可視需求調整

2. **抽卡畫面**
   - 主要遊戲畫面
   - 顯示可開啟的卡片包

3. **完成畫面**
   - 當所有卡片開完後顯示
   - 呈現慶生畫面（內容待後續討論）

#### 核心功能
1. **抽卡系統**
   - 一包卡片對應一個禮物
   - 可以開啟多包卡片（包數待確認）
   - 每包卡片開啟後會揭示對應的禮物
   - 追蹤已開啟的卡片數量
   - 判斷是否所有卡片都已開啟

2. **卡牌展示**
   - 卡牌翻轉/揭示動畫
   - 卡牌排列展示（多包時）
   - 禮物內容展示

3. **視覺效果**
   - 開卡包動畫
   - 卡牌揭示動畫
   - 禮物揭示特效（待確認）

4. **完成狀態處理**
   - 檢測所有卡片是否已開啟
   - 自動跳轉至慶生畫面

#### 待確認項目
- 總共有多少包卡片（禮物數量）
- 獎品內容與種類
- 是否需要多包連續抽卡功能
- 卡片的視覺設計風格

## 技術規格

### 前端框架
- **Vue 3** - 使用最新版本的 Vue.js
- **TypeScript** - 使用 TypeScript 進行型別檢查與開發
- **Composition API** - 使用 `<script setup>` 語法與 Composition API 風格開發

### 建置工具
- **Vite** - 使用 Vite 作為開發伺服器與建置工具

### 狀態管理
- **Pinia** - 使用 Pinia 進行狀態管理
- 遵循 Composition API 風格的 setup stores 寫法

### 路由
- **Vue Router** - 使用 Vue Router 進行路由管理

### 開發工具
- **ESLint** - 程式碼檢查
- **Prettier** - 程式碼格式化
- **Vitest** - 單元測試框架
- **Vue Test Utils** - Vue 元件測試工具

### 開發規範
- 所有元件需使用 `<script setup lang="ts">` 語法
- 狀態管理使用 Pinia 的 setup stores 寫法
- 遵循現有專案架構與檔案組織方式
- 遵循 TypeScript 型別規範
